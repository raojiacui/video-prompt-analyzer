# Video Prompt Analyzer

视频镜头分析 & AI 视频生成提示词反推工具

## 功能特点

- **视频加载** - 支持加载视频文件，自动提取关键帧
- **图片加载** - 支持直接加载多张图片进行分析
- **多 AI 支持** - 支持智谱AI / Google Gemini / OpenRouter API
- **专业分析** - 6维度深度分析（主体/场景/镜头/光影/风格/氛围）
- **结构化输出** - 自动生成可直接用于 AI 视频生成的提示词
- **结果导出** - 支持 TXT、MD、JSON 格式导出

## 支持的 AI 模型

| 服务商 | 模型 | 特点 |
|--------|------|------|
| 智谱AI | glm-4v | 国内访问快，中文理解好 |
| Google | gemini-3-flash-preview | 推理强，支持多模态 |
| OpenRouter | 多模型聚合 | 灵活切换，支持 Claude/GPT-4 等 |

## 安装使用

### 方法一：双击启动（推荐）

1. 双击 `启动.bat`
2. 首次运行会自动创建虚拟环境并安装依赖
3. 等待应用启动

### 方法二：手动安装

```bash
# 创建虚拟环境
python -m venv venv

# 激活虚拟环境 (Windows)
venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 运行
python main.py
```

## API 配置

### 在应用内配置（推荐）

1. 启动应用后，点击 **⚙️ API 设置** 按钮
2. 输入你的 API Key
3. 点击保存

### 或直接编辑配置文件

编辑 `config.json`：

```json
{
  "api_key": "你的API密钥",
  "proxy_host": "",
  "proxy_port": ""
}
```

### API 密钥获取

| 服务 | 获取地址 | 密钥格式 |
|------|----------|----------|
| 智谱AI | https://open.bigmodel.cn/usercenter/apikeys | 包含点号 |
| Google Gemini | https://aistudio.google.com/app/apikey | AIza 开头 |
| OpenRouter | https://openrouter.ai/keys | sk-or-v1 开头 |

程序会根据密钥格式自动识别服务商。

## 使用流程

### 1. 加载素材
- 点击 **📹 加载视频** 选择视频文件
- 或点击 **🖼️ 加载图片** 选择多张图片

### 2. 调整设置
- **提取帧数**：1-30 帧，帧数越多分析越细致但速度越慢
- **分析模式**：
  - **逐帧分析**：每帧单独分析生成提示词
  - **批量对比**：多帧统一分析，提取共同风格和运镜规律

### 3. 开始分析
- 点击 **🔍 开始分析** 按钮
- 等待 AI 分析完成

### 4. 查看结果
- 右侧标签页查看分析结果
- 包含画面深度描述和结构化提示词

### 5. 导出结果
- 点击 **💾 导出结果** 保存分析结果

### 批量分析输出

批量分析会额外输出：
- **镜头运动分析**：运镜方式、速度、轨迹
- **视觉风格统一分析**：跨帧的风格一致性
- **逐帧关键差异**：每帧的变化点

## 分析维度说明

| 维度 | 说明 | 优先级 |
|------|------|--------|
| 主体 | 人物/物体特征、动作、服装 | ★★★★★ |
| 场景 | 场景类型、时段天气、空间深度 | ★★★★☆ |
| 镜头 | 拍摄角度、景别、运镜、构图 | ★★★★★ |
| 光影 | 光源类型、方向、光比、色温 | ★★★★☆ |
| 风格 | 视觉风格、色彩体系、质感 | ★★★☆☆ |
| 氛围 | 情感基调、叙事暗示、节奏 | ★★★☆☆ |

## 常见问题

**Q: API 调用失败？**
A:
1. 检查 API Key 是否正确
2. 检查网络连接
3. 如使用国外 API，确保代理已开启

**Q: 视频无法加载？**
A: 确保视频格式为支持的格式（MP4/MOV/AVI/MKV/WebM）

**Q: 分析速度慢？**
A: 可以减少提取帧数，或使用批量模式

**Q: 如何切换不同的 AI 服务？**
A: 在 API 设置中直接粘贴对应服务的 API Key，程序会自动识别

**Q: OpenRouter 报错 "No endpoints found"？**
A: 检查模型名称是否正确，或到 https://openrouter.ai/models 查询可用模型
